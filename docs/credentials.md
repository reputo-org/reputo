# Credentials inventory

Every credential the platform consumes, where it lives, and which environments need it. Use this as the canonical checklist when onboarding a new dev, rotating a secret, or handing the platform to a successor.

## Conventions

- `dev` is the local stack (`make up`). Defaults are in `docker/env/examples/*.env.example`. Sensitive defaults are stand-ins; real keys are only needed for the features that hit external APIs.
- `staging` / `production` runtime values live inside Komodo today (`Settings > Variables`). The shells are declared in [`komodo/resources/variables.toml`](../komodo/resources/variables.toml).
- For a self-hosted successor, the same values go into [`docker/env/examples/prod.env.example`](../docker/env/examples/prod.env.example). See [handoff guide](handoff/README.md).

## Storage (S3 / MinIO)

| Key | dev | staging | production | Owner |
| --- | --- | --- | --- | --- |
| `AWS_REGION` | `us-east-1` | per-env | per-env | Platform |
| `S3_ENDPOINT` | `http://minio:9000` | empty (real AWS) | empty (real AWS) | Platform |
| `S3_FORCE_PATH_STYLE` | `true` | `false` | `false` | Platform |
| `STORAGE_BUCKET` | `reputo-snapshots` | per-env | per-env | Platform |

`AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` are **never read by application code**. The S3 SDK resolves them from the standard AWS credential chain:

- **dev**: `docker/env/api.env` and `docker/env/workflows.env` set local MinIO defaults. Override them there when using another S3 target.
- **preview**: docker-compose sets them on the api/worker containers (pointing at the MinIO sidecar).
- **staging/production**: attach an IAM role to the host. The SDK picks credentials up through `EC2InstanceMetadataServiceV2` or container credentials. As a fallback, set `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY` in the host shell before `make selfhost-up`.

## Komodo (only for our deploy plane, not the self-host path)

| Key | Owner | Notes |
| --- | --- | --- |
| `KOMODO_PASSKEY` | Platform | Shared between Komodo Core and each Periphery agent. |
| `KOMODO_WEBHOOK_SECRET` | Platform | HMAC secret for GitHub → Komodo webhooks; must match `secrets.KOMODO_WEBHOOK_SECRET` in the GitHub `staging` and `production` environments. |
| `KOMODO_DISCORD_WEBHOOK_URL` | Platform | Discord channel for the `reputo` alerter. |

| `KOMODO_GITHUB_OAUTH_ID` / `_SECRET` | Platform | GitHub OAuth app for Komodo login. |

A successor operating without Komodo does not need any of these.
